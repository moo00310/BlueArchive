서버 메일 보상 시스템

데디케이트 서버가 보상 메일을 등록하면, 접속 중인 클라이언트에게 즉시 배포하고
오프라인이었던 플레이어는 기간 내 재접속 시 자동으로 수신하는 시스템.

파일별 역할

클래스 실행 위치 단일 책임
BAGameModeBase 서버 전용 메일 등록 · 배포 · 수령 검증 · ClaimedMap 관리
BAPlayerController 서버 + 클라이언트 RPC 브릿지 · UID 등록
BAMailSubsystem 클라이언트 전용 수신함 보관 · 보상 요청 · 로컬 보상 적용
BAResourceSubsystem 클라이언트 전용 실제 재화 증감 · SaveGame 저장

어필 포인트

  • 보안 서버 권위(Server Authority) 기반 3단계 검증 — MailId 존재 확인, 만료 여부, 중복 수령 방지. 클라이언트가 bClaimed를 조작해도 서버 ClaimedMap에서 차단.
  • 설계 오프라인 플레이어 재접속 처리 — 단순 Multicast로 끝내지 않고 ActiveMailList를 서버에 보관해 기간 내 재접속 시 자동 수신.
  • 성능 O(1) 역방향 맵 — UIDToController + ControllerToUID 양방향 맵으로 선형 탐색 제거.
  • 문제해결 UGameInstanceSubsystem의 GetWorld() 한계 — PIE에서 null 반환 문제를 TWeakObjectPtr 기반 LocalPC 등록으로 해결.
  • 구조 관심사 분리(SoC) — GameMode는 서버 로직만, MailSubsystem은 클라이언트 수신함만 담당. 레이어 간 의존성 최소화.
